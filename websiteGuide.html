Starting up

Making your first file

Now that you have a repository, you can make your first file:

Press the icon and make a new file with the title "title.html", in this case, I will name it home.html

Next we will add some Elements into the file:

  • a <!DOCTYPE html> element
  • a <html lang="en"> element
  • a <head> element containing a <meta charset="UTF-8"/> element, a <meta name="viewport" content="width=device-width, initial-scale=1.0"/> element, and a <title> element
  • a <body> element

The <!DOCTYPE html> element will define the Html file, so that the browser knows that this is a html file

The <html lang="en"> element will say that the language is english

The <head> element contains information and linked files that help the page run

The <meta charset="UTF-8"/> element says we are using the "UTF-8" character set so that the text we want to show up on the site will display properely

The <meta name="viewport" content="width=device-width, initial-scale=1.0"/> element will make the website show up on the whole screen.

The <body> element will contain other Elements the contain the actual information the page shows you


Next, we will go look at some website basics and the other Elements we will be using to build our site.

Website basics:

First, we are going to go over some key terms that you will need to understand in order to build websites yourself.


Frontend vs Backend:

When someone asks you about the "Frontend" of a website, they are talking about all of the outward information we see. For example, the look of the website, the words on the page, the buttons we click, etc.

However, the "Backend" is to do with the data moving behind the site, when you enter your data into a form and it is submitted, what the website does with the data afterwards, is the "Backend"


Common units:

When deciding the sizes of your elements, you will run into 'px', 'vh', 'vw', etc.

The only one we are going to care about right now, is 'px' meaning pixels on your screen, this is called an 'absolute' unit, since the element will stay the same amount of pixels, no matter the screen you're on.

If you DO want to look at the other size units out of curiosity, I thought this guide was pretty thorough with it.


For colours, you have rgb, rgba, hexcodes and more.

I'm going to use Hexcodes for this website, they will look like this #ffffff (this is white) with 6 letters and numbers that decide the colour


Languages:

Now for the Frontend, we will usually be using Html, CSS and Javascript

There are other languages you can use, but these three are the most common so we will be focusing on these.

Html:

This stands for HyperText Markup Language

Stores the actual data being shown on the website, people ususally call it the "bones" of a page

It is coded with "Elements" that look like this <p> </p>, we used them to set up our file, and we'll use them to add more stuff to our site

CSS:

This stands for Cascading Style Sheets

This is used specifically for the look of our website, and will manipulate any html present

You'll see it written as some html element, a class, or id (we'll talk about these two later) and underneath a pair of { } with features inside, each of these features will have a colon after the feature and a semi-colon after the colour or size

Adding some CSS

First, you are going to create a new file called style.css in the same folder as your html file

Here, you will style your tags, however you want. I really like the colour blue, so I made everything blue!

When adding styles to your tags, color usually relates to the colour of the words/line you are applying to, the style closest to the tag (like styles on body and h1, both applying to h1) will take priority.

Add whatever stylings you want, then we need to link your CSS file to your Html file

Linking your CSS file

Go back into your Html file, and go into the head tag

Here, you are going to add the link element, this is going to let the Html page know that it should use style.css as its 'stylesheet' (where the html page should get its stying from)

Once you've done that, make sure you save both of your files, then test it

Deployment

We are going to deploy our website using vercel.

First you'll need to make sure you have all of your files saved and committed to your github repo, since vercel will be using these to get the site up

Next, create an account with vercel, make sure you select a hobby plan, and go onto the "add new" button, and press "project"

Here, select the repository that you've stored your website files in, and import, I'll be doing personal website.

Then, on the next screen, make sure that your new project page looks similar to mine, most importantly the application present and root directory, if that's the case, press deploy.

You should get this screen next!

Now, providing your html file was not called index.html, you will encounter an error because vercel couldn't find your file.

Other deployment sites will have requirements like this too, and its import that you learn this early

The way I think of it, imagine you go to another country with electronics from home, you need all of the plugs to either already suit the sockets of that country, or use an adaptor. Kind of like adapting your project to whatever site you use to deploy your website.

So, change the name of your html file to index, commit, and check your website.

And Well Done!! It should be working now, for anyone to see! You've made your first website.

What if it's still not working?

Well that's not good, first, double check all of your file names, and the endings of them, you might have misspelled them.

Then if it's still not working, check that your head tag has everything needed.

If you are still lost, I would check the Vercel Docs and if these are a bit confusing, there's no shame in asking AI, but make sure you're actually learning from it, not just blindly typing things in.
